SCSS MIXINS

Description

A bunch of utilities that I usually use on my projects:

  • Responsive grid system based on mixins, so I can keep my HTML code semantic and clean. No more -grid-medium-1-12 classes on your code, YAY!
  • Infinite number of columns, rows with 12 columns, 3 or 24 are perfectly fine
  • Allows offsets for pushing the cells to the right position
  • Utilities such as hiding and showing elements based on the size of the screen
  • Mixins for adding blocks of css to specific screen sizes
  • No gutter, every cell can have their own padding. I find it easier and more flexible this way.
  • Built with a mobile first approach, every breakpoint add styles from itself and up

@mixin row($padding:'auto')
ParamRequiredMeaning
$paddingNoPadding of the box, admits short forms like '0 25px'. If unspecified then no padding will be added
@mixin grid-cell($size:1, $of:1, $offset:0, $padding:'auto')
ParamRequiredMeaning
$sizeNoSize of the cell. Defaults to 1.
$ofNoMaximum number of cells in the row. Defaults to 1
$offsetNoNumber of rows that the cell will be pushed to the right. Defaults to 0
$paddingNoPadding of the box, admits short forms like '0 25px'. If unspecified then no padding will be added
@mixin $screenSize-grid-cell($size:1, $of:1, $offset:0, $padding:'auto')
ParamRequiredMeaning
$screenSizeYesMinimum size of the screen for the rule to be applied to. Possible values ['small', 'medium', 'large', 'x-large']
$sizeNoSize of the cell. Defaults to 1.
$ofNoMaximum number of cells in the row. Defaults to 1
$offsetNoNumber of rows that the cell will be pushed to the right. Defaults to 0
$paddingNoPadding of the box, admits short forms like '0 25px'. If unspecified then no padding will be added
@mixin $screenSize-hide()
ParamRequiredMeaning
$screenSizeYesMinimum size of the screen for the element to be hidden. Possible values ['small', 'medium', 'large', 'x-large']
@mixin $screenSize-show($display:block)
ParamRequiredMeaning
$screenSizeYesMinimum size of the screen for the element to be hidden. Possible values ['small', 'medium', 'large', 'x-large']
$displayNoType of the display for the Element. Defaults to 'block'
@mixin $screenSize-styles(){ @content }
ParamRequiredMeaning
$screenSizeYesMinimum size of the screen for the element to be hidden. Possible values ['small', 'medium', 'large', 'x-large']
@contentYesBlock of CSS rules to be applied to the element
